Introduction

L'internet, c'est le réseau qui permet de communiquer entre ordinateurs. Sur ce réseau, on trouve plusieurs services:
Le Web(consulter des pages, obtenir des informatios), Les e-mails(échanger des massages), Les newsgroups(communiquer dans des forums), Le FTP(échanger des fichiers)... Ces services convergent vers le Web. On confond souvent le Web avec ces services, car on passe à travers cette passerelle(جسر), le Web pour accéder à ses services. Le Web sert donc comme la porte d'entrée à la plupart de ses services aujourd'hui. c'est le cloud(en français l'informatique en nuage ou encore l'infonuagique) qui représente les services qui passe à travers le web. Le Web sert alors de passerelle à de nombreux services, sous la dénomination "cloud" qui est un logiciel que l'on utilise à travers une interface web. C'est le fait de stocker des fichiers à travers une interface web. Les web-emails sont des interfaces web qui permettent de consulter les e-mails.


Page Web

Une page web est un document simple qui peut être affiché par un navigateur web (tel que Mozilla Firefox, Google Chrome, Internet Explorer Edge ou encore Safari...) Ce document est écrit à l'aide du langage HTML, et peut inclure diverses autres ressources telles que:
1- des feuilles de style qui déterminent l'apparence de la page
2- des scripts qui ajoutent des fonctions interactives
3- du contenu multimédia images, sons, vidéos...
Le terme page web désigne spécifiquement des documents HTML. Toutes les pages web sont associées à une adresse unique. Pour atteindre une page web, il suffit d'entrer son adresse dans la barre d'adresse du navigateur.


Hypertexte et L'hyperlien

L'hypertexte est un texte contenant des liens vers d'autres textes, par opposition à un unique flux linéaire comme dans un roman.
Le terme a été inventé par Ted Nelson aux alentours de 1965.
Un hypertexte est un document ou un ensemble de documents contenant des unités d'information liées entre elles par des hyperliens. Ce système permet à l'utilisateur d'aller directement à l'unité qui l'intéresse.
Un hyperlien ou lien hypertexte ou lien web ou simplement lien, est généralement sous la forme de texte cliquable, qui permet le passage d'une page Web à une autre à l'aide d'un clic.Il permet de passer automatiquement d'un document consulté à un document lié. Les hyperliens sont notamment utilisés dans le World Wide Web pour permettre le passage d'une page Web à une autre à l'aide d'un clic.


Le site web

Un site web ou simplement site est un ensemble de pages web et de ressources associées qui partagent un nom de domaine. Ces pages web sont regroupées(reliées) entre elles de différentes façons(manières). Chaque page d'un site fournit des liens explicites, généralement sous la forme de texte cliquable, qui permettent à l'utilisateur de naviguer entre les pages du site web. Elle est composé de nombreux fichiers sous forme de contenu textuel(code), comme feuilles de styles, contenus média etc... Lors de la construction d'un site web, ces fichiers doivent être organisés et rangés dans un dossiers, afin qu'ils puissent intéragir les uns avec les autres et que le contenu s'affiche correctement. Une fois que ceci est fait, on peut alors téléverser ces fichiers sur un serveur. Pour atteindre un site web, on doit saisir son nom de domaine dans la barre d'adresse du navigateur. Le navigateur affichera alors la page principale, appelée page d'accueil du site web.


Serveur Web !?

Un serveur web est un ordinateur qui héberge un site web ou plusieurs sites web sur Internet. le travail du serveur est de recevoir des requêtes, de générer les pages web et de les envoyer aux clients.
Héberger: signifie que toutes les pages web et fichiers associés sont localement enregistrés sur cet ordinateur. À la demande d'un utilisateur, le serveur web transmettra la page web du site web hébergé au navigateur de l'utilisateur. Il peut désigner un matériel ou un logiciel, ou les deux à la fois.
Du côté matériel, un serveur Web est un ordinateur qui stocke le logiciel de serveur Web et les fichiers composants un site Web (des documents HTML, des images, des feuilles de style CSS et des fichiers JavaScript...). Il est connecté à l'Internet et prend en charge l'échange de données physiques avec d'autres périphériques connectés au Web.
Du côté logiciel, un serveur Web comprend plusieurs composants qui contrôlent la manière dont les utilisateurs Web accèdent aux fichiers hébergés, au minimum un serveur HTTP . Un serveur HTTP est un logiciel qui comprend les URL (adresses Web). HTTP est le protocole utilisé par le navigateur pour afficher les pages Web. Il est accessible via les noms de domaine des sites Web qu'il stocke et transmet leur contenu au périphérique de l'utilisateur final.
Au niveau le plus élémentaire, chaque fois qu'un navigateur a besoin d'un fichier hébergé sur un serveur Web, il le demande via HTTP. Lorsque la demande parvient au bon serveur Web, le serveur HTTP accepte la demande, trouve le document demandé et le renvoie au navigateur, également via HTTP. Si non, une réponse 404 est renvoyée au navigateur.
On distingue deux types de serveurs: serveur Web statique ou dynamique.
Un serveur Web statique , ou pile, consiste en un ordinateur avec un serveur HTTP. Nous l'appelons "statique" car le serveur envoie ses fichiers hébergés sans les générer, tels quels les a au navigateur.
Un serveur Web dynamique comprend un serveur Web statique et un logiciel supplémentaire, généralement un serveur d'applications et une base de données . Nous l'appelons "dynamique" car le serveur d'applications met à jour les fichiers hébergés avant de les envoyer au navigateur via le serveur HTTP.
Par exemple, pour produire les pages Web finales que l'on voit dans le navigateur, le serveur d'applications peut remplir un modèle HTML avec le contenu d'une base de données.
Pour récupérer une page Web, le navigateur envoie une demande au serveur Web, qui procède ensuite à la recherche du fichier demandé dans son propre espace de stockage. Lorsqu'il trouve le fichier, le serveur le lit, le traite au besoin et l'envoie au navigateur.


Le protocole HTTP

HTTP: HyperText Transfer Protocol: protocole de transfert hypertexte, c'est un protocole d'application définissant le language de communication entre les clients et les serveurs. C'est le langage qu'utilisent le client et le serveur pour s'échanger des informations. C'est une "langue" de communication entre le client et le serveur.
Voici comment un client demande une page web à un serveur: 

  GET /page.html HTTP/1.0 Host:GET /page.html HTTP/1.0
  Host: example.com
  Referer: http://example.com/
  User-Agent: CERN-LineMode/2.15 libwww/2.17b3 example.com Referer: http://example.com/ User-Agent: CERN-LineMode/2.15 libwww/2.17b3

Le serveur va ensuite travailler en utilisant un langage serveur, puis renvoie la page HTML avec une réponse comme ceci: 

  HTTP/1.0 200 OK
  Date: Fri, 31 Dec 1999 23:59:59 GMT
  Server: Apache/0.8.4
  Content-Type: text/html
  Content-Length: 59
  Expires: Sat, 01 Jan 2000 00:59:59 GMT
  Last-modified: Fri, 09 Aug 1996 14:21:40 GMT

Cette réponse indique que tout s'est bien passé : le code "200 OK" signifie que la page existe et que le serveur peut la renvoyer.
Il existe de nombreux codes qui permettent au serveur d'indiquer au client ce qui s'est passé, par exemple le code "404", signifie que la page n'a pas pu être trouvée sur le serveur. Le code "302" signifie que la page a changé de place. En général, le serveur redirige le client vers la nouvelle adresse de la page.
Un serveur HTTP est souvent associé au port 80 de la machine hôte. Voici quelques exemples de serveurs HTTP.

  • Apache HTTP Server : classique
  • Apache TomCat : évolution pour Java JEE
  • IIS (Internet Information Services) : Microsoft
  • Node.js : codé en JavaScript.

Un serveur Web est un serveur HTTP. On parle de "baie de serveurs" pour désigner la colonne où tous les serveurs sont assemblés (on dit qu'ils sont "rackés"). Les serveurs sont entassés les uns sur les autres.
On parle de "centre de données" (datacenter), c'est donc le lieu où les données des sites web sont stockées et analysées.
Un datacenter est un lieu où sont situés, stockés habituellement les serveurs, Ce sont des bâtiments très sécurisés. les serveurs sont souvent des machines "plates", pour pouvoir les entasser plus facilement. Il n'y a pas d'écran, car personne ne les utilise directement en général. Ils se contentent de transmettre le contenu des sites web sur le réseau.
Ils sont reliés à un câble de fibre optique permettant un échange ultrarapide. On retrouve cette fibre souvent enterrée sous terre, mais aussi au fond de la mer! En fait, la majorité du trafic d'Internet passe par des câbles sous-marins.


Les protocoles

Comment font les ordinateurs pour communiquer entre eux, quelle langue parlent-ils entre eux. On a inventé des langages de communication pour que les machines se parlent entre elles. On les appelle les protocoles.
On les distingue à différents types de niveau: Protocoles bas niveau TCP UDP et Protocoles haut niveau
Les Protocoles bas niveau sont utilisés de façon très globales sur internet, on les retrouve partout. Vint Cerf a inventé le protocole TCP/IP. Il est donc considéré comme le père d'Internet!. C'ets lui qui a inventé la façon dont deux ordinateurs peuvent communiquer entre eux, avec n'importe quel type d'information. Aujourd'hui TCP/IP est utilisée pour faire transiter des pages web, des e-mails, des newsgroupes, des vidéos... En fait la quasi-totalité des informations d'internet utilise TCP/IP pour communiquer.
Les Protocoles haut niveau sont généralement basés sur TCP (ou UDP). Ils forment une surcouche qui englobe TCP.
HTTP: le protocole qui permet d'échanger des pages web entre le client et le serveur. En plus du langage HTML, Tim Berners-Lee a inventé les bases du protocole HTTP qui permet d'échanger les pages. C'est ce que signifie le "http://" que vous voyez au début des adresses web!
HTTPS: identique à HTTP, avec le "S" en plus qui signifie "Secure". Les pages sont cryptées sécurisées pour garantir que personne ne peut les lire (par exemple quelqu'un qui espionnerait les données qui transitent dans les câbles sous-marins!).
FTP: signifie File Transfer Protocol, protocole de transfert de fichiers. Il permet d'échanger des fichiers.
SMTP: permet d'envoyer des e-mails. Etc...

On peut se baser sur un protocole pour faire plus simple encore plus facile à utiliser pour communiquer entre des ordinateurs.
