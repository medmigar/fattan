Fichier HTML

Itroduction

La page web est un document HTML, qui est lui même un fichier avec l'extension .html. HTML c'est le langage de balisage standard pour la création des pages Web. Le language HTML sert à décrire la structure de la page web que le navigateur doit afficher. IL se compose d'une série d'éléments HTML:les balises qui indiquent au navigateur comment afficher le contenu. Le navigateur n'affiche pas les balises HTML, mais il les utilise pour déterminer comment afficher le contenu du document. Depuis les débuts du World Wide Web, il existe de nombreuses versions de HTML. Dans ce cours on va suivre la dernière norme HTML5.


Structure simple de la page web

      <!DOCTYPE html>
      <html>
        <head>
         <title>Titlre</title>
         <meta charset="utf-8">
         <link rel="stylesheet" href="">
         <script src=""></script>
        </head>
        <body>
         <h1>Mon premier titre</h1>
         <p>Mon premier paragraphe.</p>
         <a href="https://www.google.com">Ceci est lien</a>
         <img src="med.jpg" alt="photo Med" width="104" height="142">
        </body>
      </html>

Un élément HTML est défini par une balise de début, du contenu et une balise de fin.
Remarque: certains éléments HTML n'ont pas de contenu, comme l'élément <meta charset="utf-8">. Ces éléments sont appelés éléments vides. Les éléments vides n'ont pas de balise de fin !

  • La déclaration: <!DOCTYPE html> indique que ce document est un document HTML5. IL indique au navigateur qu'il s'agit du code HTML5.
  • L'élément <html> est l'élément racine de la page HTML, avec l'attribut lang.
  • L'élément <head> contient des méta-informations sur la page HTML. Le contenu de cet élément n'est pas visible par le visiteur.
  • L'élément <title> spécifie un titre pour la page HTML, qui s'affiche dans la barre du titre du navigateur ou dans l'onglet de la page. Il est important de le définir pour plusieurs raisons. Le titre est utilisé par les moteurs de recherche
  • La balise <meta> indique des informations de métadonnées qui ne sont généralement pas visibles par le visiteur, ells représentent les informations descriptives sur la page web. Le jeu des caractères charset="utf-8" peut sembler sans importance, mais il est essentiel pour définir comment les ordinateurs interprètent les caractères.
  • L'élément <link> permet de mettre en relation la page avec d'autres documents externes. La plupart du temps cela concerne les feuilles de style CSS externes avec une relation du type stylesheet, mais il est aussi possible de définir d'autres types de relations, par exemple avec author, icon, prefetch, next, prev..etc.
  • L'élément <script> permet d'ajouter des scripts de JavaScript qui vont s'éxécuter côté client dans le navigateur dès leur chargement.
  • L'élément <body> définit le corps du document. Il est le conteneur pour tous les contenus visibles, tels que les titres, les paragraphes, les images, les hyperliens, les tableaux, les listes...etc.
  • L'élément <h1> définit un grand titre
  • L'élément <p> définit un paragraphe
  • L'élément <a href="https://www.google.com">Google</a> définit un lien. La destination du lien est spécifiée dans l'attribut href. L'attribut href spécifie l'URL, absolue ou relative de la page vers laquelle dirige ce lien. Rien n'empêche bien sûr la balise <a> de contenir dautres balises.
  • Les attributs sont utilisés pour fournir des informations supplémentaires aux éléments HTML. Les attributs sont toujours spécifiés dans la balise de début Ils se présentent généralement sous forme de paires nom/valeur comme: nom = "valeur"
  • L'élément <img src="med.jpg" alt="photo Med" width="104" height="142">définit une image. La destination de l'image est spécifiée dans l'attribut src:source. L'attribut src spécifie le chemin d'accès à l'image à afficher. Elle a deux attributs obligatoires: src: l'URL, absolue ou relative de l'image
    alt: une description textuelle de l'image, obligatoire pour les situations où l'image ne peut pas être affichée

Les autres éléments HTML5

Une fois la structure de base est préparée, on commence à placer d'autres éléments HTML pour le contenu selon le besoin comme: paragraphes, listes, liens, images, vidéos, tableaux de données, formulaires...etc. Éléments de section <section>, <article>, <header>, <footer>, <nav> et <aside> Élément <datalist> Éléments <details> et <summary> Éléments <figure> et <figcaption> Éléments média <audio> et <video> complétés par <source> Élément <canvas> Éléments <meter>, <output> et <progress> Élément <time> et <mark>
Balises historiques, Elles sont encore valides en HTML5:
<b>: texte en gras <i>: texte en italique <u>: texte souligné <br>: saut de ligne
Listes
Liste non ordonnée <ul> <li>sucre</li> <li>céréales</li> <li>lait</li> </ul>
Liste ordonnée <ol> <li>sucre</li> <li>céréales</li> <li>lait</li> </ol>
Remarque: On note que:
<ul> signifie unordered list, <ol> signifie ordered list, <li> signifie list item,
Vidéo: En plus des images, HTML5 permet d'inclure des vidéos, grâce à la balise <video>.
<video src="_static/Chrome_ImF.webm"> Ici, une vidéo représentant... </video>
Le contenu textuel de la balise <video>, s'affiche uniquement si le navigateur ne supporte pas cette balise. Il joue le rôle de l'attribut alt pour les images.
Attributs de la balise video
La balise video supporte les attributs suivants:
autoplay : lance la lecture automatiquement, controls : affiche l’interface de lecture, loop : reprend automatiquement la lecture à la fin de la vidéo, muted : fixe le volume sonore à 0, poster : URL d’une image à afficher avant la lecture
Méta-données
Les balises <meta> sont situées dans l’élément <head> du document. Elles contiennent des méta-données : des données sur le document lui-même.
<meta charset="utf-8"> <meta name="description" lang="en" content="apprenez plus de tours en HTML"> <meta name="author" content="Pierre-Antoine Champin"> <meta name="keywords" content="html web iut">
Formulaires et interactivité
Champs de saisie Certaines pages HTML comportent des champs permettant à l'utilisateur de saisir de l'information. La balise offrant cette possibilité est la balise <input>.
Types de champs
HTML offre un certain nombre de types de champs, les plus utilisés étant:
<input type="text">, <input type="password">, <input type="checkbox">, <input type="radio">, <input type="file">, <input type="number">, <input type="range">;
Liste
Un autre type de champ permet de sélectionner un ou plusieurs éléments dans une liste. L'attribut multiple autorise la sélection de plusieurs éléments.
<select> <option>Georges</option> <option>John <option>Paul</option> <option>Ringo</option> </select>
Zone de texte
Le champ <input> ne permet pas la saisie de texte sur plusieurs lignes. Pour cela, on utilise le champ <textarea>.
<textarea placeholder="type your message">Hello world</textarea>
Boutons
La balise <button> permet de créer des boutons. <button>Ok</button> <button><em>Annuler</em></button>
Historiquement, les boutons étaient générés par la balise <input> avec des types spécifiques: button et submit notamment.


Éditeurs HTML

Un simple éditeur de texte est suffisant pour écrire et modifier un fichier HTML. Cependant il est recommandé de choisir un éditeur de texte professionnel. Mais l'utilisation d'un simple éditeur de texte est un bon moyen pour les débutants.


Structure minimale de base de la Page Web

Tous les documents HTML doivent commencer par la déclaration de type de document: <!DOCTYPE html>.
Le document HTML lui-même commence par <html>et se termine par </html>.
La partie visible du document HTML est comprise entre <body>et </body>.
Voici un exemple de page minimale: 

        <!DOCTYPE html>
        <html>
          <body>
            <h1>Mon premier titre</h1>
            <p>Mon premier paragraphe.</p>
          </body>
        </html>

Comme on voit le HTML utilise ce que l'on nomme des balises afin de structurer les informations.


Navigateurs Web

Le navigateur Web est le logiciel qui sert à surfer sur la Toile en affichant les pages Web. Le but d'un navigateur Web (Chrome, Edge, Firefox, Safari) est de lire les documents HTML et de les afficher correctement.
Un navigateur n'affiche pas les balises HTML, mais les utilise pour déterminer comment afficher le document. Tous les ordinateurs actuels, qu'ils soient fixes, portables ou mobiles, intègrent au moins un navigateur Web.


Principales parties d'un document

La page HTML peut être divisé en deux parties:
L'en-tête située entre les balises <head> et </head>, qui regroupe toutes les méta-informations.
La deuxième partie comprend le corps <body>,ou on retrouvera tout le reste du contenu HTML, structuré par des balises variées. Pour mettre en œuvre le marquage sémantique, HTML fournit des balises dédiées que l'on peut utiliser pour représenter les différentes parties d'un document qui sont:
En‑tête (header), Barre de navigation, Contenu principal, Barre latérale et Pied de page C'est un endroit pour mettre de l'information commune (comme l'en-tête)
header : <header>. Une grande bande placée en haut de la page avec un titre ou un logo. C'est là où les principales informations du site restent d'une page à l'autre.
barre de navigation : <nav>.
contenu principal : <main>, avec diverses sous‑sections de contenu représentées à l'aide de des éléments <article>, <section> et <div>.
barre latérale : <aside> ; souvent mise à l'intérieur de l'élément <main>.
pied de page : <footer>.C'est à cet endroit que l'on peut mettre des informations commune comme à l'en-tête.


Les balises structurantes de HTML5

HTML5 a introduit des balises nouvelles pour structurer et bien construit les pages Web, et pour être prêt à être mis en forme ensuite. Ses balises sont:
<header>: l'en-tête. Il peut contenir tout ce que l'on veut: images, liens, textes…etc.
<footer>: le pied de page À l'inverse de l'en-tête, le pied de page se trouve en général tout en bas du document. Dans le pied de page on trouve des informations comme des liens de contact, le nom de l'auteur…etc.
<main>: contenu principal de la page
<nav>: principaux liens de navigation La balise <nav> doit regrouper tous les principaux liens de navigation du site. On placere souvent le menu principal du site dans cet élément.
<section>: une section de page. La balise <section> sert à regrouper des contenus en fonction de leur thématique. Elle englobe généralement une portion du contenu au centre de la page.
<aside>: informations complémentaires. La balise <aside> est conçue pour contenir des informations complémentaires au document que l'on visualise. Ces informations sont généralement placées sur le côté. Il peut y avoir plusieurs blocs <aside> dans la page.
<article>: un article indépendant. La balise <article> sert à englober une portion généralement autonome de la page. C'est une partie de la page qui pourrait ainsi être reprise sur un autre site.


Exemple d'utilisation des balises

  <!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />
        <title>Zozor - Le Site Web</title>
    </head>

    <body>
        <header>
            <h1>Zozor</h1>
            <h2>Carnets de voyage</h2>
        </header>
        
        <nav>
            <ul>
                <li><a href="#">Accueil</a></li>
                <li><a href="#">Blog</a></li>
                <li><a href="#">CV</a></li>
            </ul>
        </nav>
        
        <section>
            <aside>
                <h1>À propos de l'auteur</h1>
                <p>C'est moi, Zozor ! Je suis né un 23 novembre 2005.</p>
            </aside>
            <article>                
                <h1>Je suis un grand voyageur</h1>
                <p>Bla bla bla bla (texte de l'article)</p>
            </article>
        </section>
        
        <footer>
            <p>Copyright Zozor - Tous droits réservés<br />
            <a href="#">Me contacter !</a></p>
        </footer>
        
    </body>
</html>
